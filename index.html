Installing Git

git is a distributed version control system used to track computer file changes. While it is possible to use GitHub without installing git, it is more efficient, and also more beneficial to become familiar with using it. If you do not already have it installed, you can download it for your OS at https://git-scm.com/downloads.

The Git site also contains documentation and tutorials in numerous languages for those new to it. The following YouTube videos also provide useful information for beginners:

  • Git Tutorials (Corey Schafer)
  • Git and GitHub for Beginners - Crash Course (Gwendolyn Faraday at freeCodeCamp.org)
  • Git & GitHub Crash Course For Beginners (Traversy Media)
Git Setup

Once git is installed, you should configure it with your user name and email. Type the following, with your name and email information substituted, at the command line (without the $)

$ git config --global user.name "John Doe" $ git config --global user.email johndoe@example.com

A file directory can be tracked with git by navigating to it (in Windows you can right-click on the folder and select 'git bash here') and intitalizing a repository by typing the following:

$ git init

This will create a master branch for your project. You also need to declare which files you want to track. You can add specific files by name, or add all files in the folder as shown below. The first command adds the files to what is called the Staging Area, and the second logs the changes in them.

$ git add -A $ git commit -m "some message explaining what I did to these files"

A GitHub account can be created on the GitHub website.

Account setup and verification is fairly straight forward.

GH Repo Creation

On GitHub, a new repository can be created by clicking the green 'new' button. If you are on the main GitHub page, it is on the top left, and looks like this:

You will be prompted to create a name for your repository. Once created, you will be given information that includes a git command and https url that you can use to associate an existing git project on your local machine with this remote GitHub repository. In your local git project directory, type or paste it in the Git command line.

$ git remote add origin https://github.com/username.projectName.git
Using gh-pages

Since we want to publish the project with gh-pages, we also need to create a gh-pages branch. Running the command below in the git directory will create this branch and switch to it.

$ git checkout -b gh-pages

This branch can be uploaded to your GitHub repository with:

$ git push origin gh-pages

You might be asked to supply your github authorization here.

It may take a few minutes for your changes to be updated on GitHub. Once they have been, you will be able to see your project at:

https://username.github.io/projectname/
